# components/
#
# 通用游戏组件 — 每个子目录是一个独立的"模块"（脚本 + 场景 + 可能的资源）
#
# 核心原则：
#   1. 每个组件通过 @export 暴露接口，不依赖其他组件
#   2. 每个组件场景可以独立 F6 运行不报错
#   3. 组件之间通过信号（signal）通信，不直接访问对方属性
#   4. 同目录下 .gd 和 .tscn 放一起
#
# 组织逻辑：
#   按功能域分大类 → 每个子功能一个独立目录 → 目录内脚本和场景平级
#   movement/   — 所有"角色怎么动"的组件
#   combat/     — 所有"怎么打"的组件
#   character/  — 所有"角色属性数值"的组件
#   world/      — 所有"世界交互"的组件
#   ui/         — 所有"界面显示"的组件
